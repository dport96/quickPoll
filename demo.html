ðŸ”§ Technical Overview

Features

  • Three Poll Types: Simple voting, rating scales (1-5 stars), and drag-and-drop ranking
  • Real-time Results: See voting results update instantly
  • Mobile Responsive: Works seamlessly on phones, tablets, and desktops
  • No Server Required: Pure client-side JavaScript with localStorage
  • Easy Sharing: Share polls via URL with query parameters
  • Email Authentication: Optional user identification for fair voting

How It Works

MyVote uses query string parameters to pass poll data between pages and localStorage to persist votes. This means:

  • No database or server setup required
  • Works as static files on any web server
  • Data persists locally in each browser
  • Polls can be shared via URL

URL Structure

Polls are created and shared using URL parameters:

?mode=vote&id=pollId&title=Poll%20Title&type=simple&opt0=Option1&opt1=Option2

Parameters:

  • mode: 'create', 'vote', or 'results'
  • id: Unique poll identifier
  • title: Poll title (URL encoded)
  • type: 'simple', 'rating', or 'ranking'
  • opt0, opt1, opt2...: Poll options (URL encoded)

Email Authentication Setup

The email authentication version includes:

  • Persistent Sign-In: Enter email once, stay signed in across polls
  • Simple email entry modal
  • One vote per email address enforcement
  • Optional email whitelist validation
  • Session management with localStorage
  • Seamless Experience: No need to re-enter email for subsequent polls

Perfect for professional surveys, team voting, or any scenario where you need to prevent duplicate votes while keeping setup simple. Users only enter their email once and stay signed in!
